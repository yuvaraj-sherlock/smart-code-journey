# 🚀 PatternForge

> A structured Java project to master Data Structures & Algorithms through **pattern recognition**, not memorization.

---

## 🧠 Vision

Most developers solve problems one by one.
This project is built to do something smarter:

👉 Identify patterns
👉 Extract reusable tweaks
👉 Apply them across problems

**Goal:** Become a *pattern-driven problem solver*, not just a coder.

---

## ⚡ Core Philosophy

* ❌ Don’t memorize solutions

* ❌ Don’t rely on random attempts

* ✅ Recognize patterns

* ✅ Apply common tweaks

* ✅ Build reusable logic

---

## 🧩 Pattern Thinking Model

Every problem is approached using:

1. **Pattern Identification**

    * Example: Set, Map, Sliding Window, Two Pointers

2. **Trigger Recognition**

    * What hint in the problem suggests the pattern?

3. **Core Tweak**

    * The reusable trick (🔥 most important)

4. **Implementation**

    * Clean, minimal Java code

5. **Reusability**

    * Where else can this be applied?

---

## 🔥 Core Patterns Covered

| Pattern        | When to Use             | Core Tweak          |
| -------------- | ----------------------- | ------------------- |
| Set            | Uniqueness / duplicates | `!set.add(x)`       |
| Map            | Counting / frequency    | `getOrDefault()`    |
| Sliding Window | Subarray / substring    | Expand + shrink     |
| Two Pointers   | Sorted data / pairs     | Slow & fast pointer |
| Stack          | LIFO / validation       | Push & pop logic    |
| Queue          | Sequential processing   | FIFO                |
